# 后台最小反馈统计设计方案

> 日期：2026-03-26

---

## 1. 本轮目标

**让管理层在后台首页一眼看到话术反馈概况（赞/踩数量、来源分布、最差话术），为话术持续优化提供第一手数据依据。**

---

## 2. 为什么要做反馈统计

| 问题 | 说明 |
|------|------|
| 通话结果不够 | 当前后台只有"接通/无意向/有意向"三个结果指标，无法知道**哪条话术好用、哪条不好用** |
| 反馈补充细粒度数据 | 点赞/点踩精确到每一句话术建议，能定位到具体哪条话术需要优化 |
| 统一 vs 自定义对比 | 可以验证门店自定义话术是否比总部统一话术更受一线欢迎 |
| 低成本高价值 | 数据已有（script_feedbacks 表），只需一个聚合查询 + 最小展示即可 |

---

## 3. 第一版建议展示的最小指标

### 概览指标（卡片形式）

| 指标 | 说明 |
|------|------|
| 今日点赞总数 | SUM(like) |
| 今日点踩总数 | SUM(dislike) |
| 统一话术点赞数 | scriptSource = global 的 like 数 |
| 统一话术点踩数 | scriptSource = global 的 dislike 数 |
| 自定义话术点赞数 | scriptSource = store 的 like 数 |
| 自定义话术点踩数 | scriptSource = store 的 dislike 数 |

### 排行指标（表格形式）

| 指标 | 说明 |
|------|------|
| 被点踩最多的话术 Top 5 | 按 messageText 聚合 dislike 数，降序取前 5 |
| 被点赞最多的话术 Top 5 | 按 messageText 聚合 like 数，降序取前 5 |

> 第一版共 6 张卡片 + 2 个表格，指标简洁，管理层一屏可看完。

---

## 4. 第一版建议展示的最小维度

| 维度 | 第一版是否展示 | 说明 |
|------|---------------|------|
| 话术来源（统一 / 自定义） | ✅ 展示 | 概览卡片已按来源分开 |
| 具体话术文本 | ✅ 展示 | Top N 表格中直接显示 messageText |
| 按门店 | ⏳ 暂不展开 | super_admin 看全局汇总即可；store_admin 自动收口到本店 |
| 按用户 | ❌ 暂不做 | 第一版不做个人维度分析 |
| 按时间趋势 | ❌ 暂不做 | 第一版只看今日 |

---

## 5. 角色查看范围建议

| 角色 | 可看范围 |
|------|----------|
| super_admin | 全部门店的反馈统计（全局汇总） |
| store_admin | 仅本门店的反馈统计（自动按 storeId 收口） |
| 普通 user | **第一版不开放**。理由：反馈统计是管理决策工具，普通用户看自己的反馈没有可操作性 |

---

## 6. 页面形态建议

**建议：直接放到现有 /admin 首页**

理由：
1. 当前 /admin 首页已有"今日通话统计"卡片，反馈统计性质相同，放在一起形成完整的运营数据看板
2. 独立页面目前数据量太小，单独一页显得空洞
3. 减少点击层级，管理层打开后台就能一眼看到
4. 后续数据量大了再考虑拆分独立页

具体位置：在现有"今日通话统计"下方，新增一个"话术反馈概览"区域。

---

## 7. 后端最小统计接口建议

### GET /api/admin/stats/feedback-today

| 项目 | 说明 |
|------|------|
| 路径 | GET /api/admin/stats/feedback-today |
| 权限 | super_admin / store_admin |
| storeId 收口 | store_admin 自动按 storeId 过滤；super_admin 看全部 |

### 返回字段

```
{
  "success": true,
  "summary": {
    "totalLikes": 12,
    "totalDislikes": 3,
    "globalLikes": 8,
    "globalDislikes": 2,
    "storeLikes": 4,
    "storeDislikes": 1
  },
  "topLiked": [
    { "messageText": "...", "count": 5 }
  ],
  "topDisliked": [
    { "messageText": "...", "count": 3 }
  ]
}
```

> 只做一个接口，不做复杂筛选参数。Top N 固定取 5 条。

---

## 8. 明确暂不做的内容

| 暂不做 | 原因 |
|--------|------|
| 趋势图 | 第一版只看今日 |
| 多时间范围筛选 | 增加接口复杂度 |
| 导出 CSV/Excel | 后续再做 |
| 复杂钻取分析 | 不是第一版目标 |
| 反馈详情审核页 | 后续再做 |
| 反馈备注分析 | 当前无备注字段 |
| 多角色复杂报表 | 保持最小权限模型 |
| 按用户维度分析 | 不是管理层关心的第一优先级 |

---

## 9. 建议的最小实现顺序

1. **后端：实现 GET /api/admin/stats/feedback-today** — 一个聚合查询接口，按角色自动收口 storeId，返回概览数字 + Top 5 榜单
2. **前端：在 /admin 首页新增"话术反馈概览"区域** — 6 张指标卡片（总赞/总踩 + 按来源拆分） + 2 个 Top 5 表格
3. **权限验证** — 确认 store_admin 只能看到本店数据，super_admin 看全局
4. **端到端验收** — 用 user1 点赞/踩若干话术 → 用 admin 登录后台 → 确认反馈统计数据正确显示
