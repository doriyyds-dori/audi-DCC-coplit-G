# 智慧外呼助手第一阶段前端实施任务清单

> 日期：2026-03-24

---

## 1. 实施原则

1. **小步改造**：每个任务包独立完成、独立验收，出问题可单独回滚
2. **优先复用**：现有 12 个可零改动复用的文件不碰，只在外围新增
3. **便于回滚**：路由、认证等新增模块不侵入现有组件内部逻辑；现有组件只做最小扩展（如新增 props），不重构内部实现
4. **先跑通再完善**：每个任务包完成后先验证功能可用，再考虑细节优化

---

## 2. 前端改造任务拆分

### 任务包 1：引入路由基础设施

- **目标**：为项目加入路由能力，不改动任何现有组件
- **涉及现有文件**：`main.tsx`（包裹 BrowserRouter）、`App.tsx`（拆为路由入口）
- **新增文件**：`src/pages/AppPage.tsx`（将现有 App.tsx 的内容搬入）、`src/pages/LoginPage.tsx`（占位）
- **用户感知**：打开 `/app` 看到话术助手（与现在一致），打开 `/login` 看到空白登录页占位，`/` 根路径重定向到 `/app`
- **验收标准**：
  - `/app` 路由正常显示话术助手，所有功能不受影响
  - `/login` 路由显示登录页占位内容
  - `/` 根路径根据登录态和角色进行重定向（普通用户 → `/app`，管理员 → `/admin`）
  - 直接访问不存在的路由重定向到 `/app`

---

### 任务包 2：登录页 UI

- **目标**：实现登录页的表单 UI，暂不接后端
- **涉及现有文件**：无
- **新增文件**：`src/pages/LoginPage.tsx`（覆盖占位版本）
- **用户感知**：打开 `/login` 看到用户名密码输入框和登录按钮，点击后控制台打印输入值
- **验收标准**：
  - 表单可输入用户名和密码
  - 空字段点击登录有提示
  - 密码输入框有遮挡
  - 登录按钮点击后控制台输出（不跳转）

---

### 任务包 3：认证上下文 + 路由守卫

- **目标**：创建 AuthContext 管理登录状态，未登录访问 `/app` 跳转到 `/login`
- **涉及现有文件**：`main.tsx`（包裹 AuthProvider）
- **新增文件**：`src/auth/AuthContext.tsx`、`src/auth/ProtectedRoute.tsx`
- **用户感知**：未登录时任何页面自动跳转到登录页；使用硬编码测试账号可登录进入话术助手
- **验收标准**：
  - 未登录访问 `/app` → 跳转 `/login`
  - 普通用户登录 → 跳转 `/app`
  - 管理员登录 → 跳转 `/admin`
  - 刷新页面不丢失登录态（Token 存 localStorage）
  - 点击登出 → 返回登录页

---

### 任务包 4：HeaderToolbar 扩展用户信息

- **目标**：顶部工具栏右侧显示当前用户姓名和登出按钮
- **涉及现有文件**：`HeaderToolbar.tsx`（新增 props：`displayName`、`onLogout`）
- **新增文件**：无
- **用户感知**：顶部右侧看到自己的名字，点击登出回到登录页
- **验收标准**：
  - 显示当前登录用户的 `display_name`
  - 点击登出清除 Token 并跳转 `/login`
  - 现有工具栏功能不受影响

---

### 任务包 5：「结束并记录通话」按钮

- **目标**：在通话面板中新增独立的「结束并记录通话」按钮，与「重置对话」明确分离
- **涉及现有文件**：`ChatPanel.tsx`（新增一个按钮和对应的 props 回调）
- **新增文件**：无
- **用户感知**：通话过程中看到两个不同按钮：「重置对话」（重新演练，不记录）和「结束并记录通话」（触发状态选择）
- **验收标准**：
  - 「重置对话」只清空对话，**不弹出状态选择**
  - 「结束并记录通话」弹出状态选择弹窗
  - 两个按钮视觉上有明确区分（颜色/位置不同）
  - 未开始通话时两个按钮均不显示

> ⚠️ **风险提醒**：这是 LOGIN_HOME_FLOW_PLAN.md 中特别强调的关键区分点。实施时必须确保「重置对话」的回调链中不触发任何记录逻辑。

---

### 任务包 6：通话状态选择弹窗

- **目标**：创建 CallStatusModal 组件，通话结束后弹出，选择状态后回调
- **涉及现有文件**：`App.tsx`（或 `AppPage.tsx`，组装弹窗与按钮的关系）
- **新增文件**：`src/CallStatusModal.tsx`
- **用户感知**：点击「结束并记录通话」后弹出弹窗，必须选择未接通/无意向/有意向，选完后对话区重置
- **验收标准**：
  - 弹窗有 3 个状态选项
  - 不选择不能关闭弹窗（无关闭按钮，点击遮罩不关闭）
  - 选择后触发回调（第一阶段先 console.log，接后端在后端任务中完成）
  - 选择后对话自动重置，可开始下一通

---

### 任务包 7：管理后台页面框架

- **目标**：创建管理后台的基础路由和页面框架，含 3 个子页面占位
- **涉及现有文件**：路由配置
- **新增文件**：`src/pages/AdminLayout.tsx`、`src/pages/AdminStats.tsx`（占位）、`src/pages/AdminUsers.tsx`（占位）、`src/pages/AdminStores.tsx`（占位）
- **用户感知**：管理员登录后进入管理后台，看到顶部导航可切换三个页面，有"进入话术助手"入口
- **验收标准**：
  - 管理员登录后跳转 `/admin`
  - 普通用户访问 `/admin` 被拦截跳转 `/app`
  - 三个子菜单可正常切换（`/admin`、`/admin/users`、`/admin/stores`）
  - "进入话术助手"链接可跳转到 `/app`

## 3. 推荐实施顺序

| 顺序 | 任务包 | 原因 |
|------|--------|------|
| 1 | 引入路由基础设施 | 所有后续任务都依赖路由存在 |
| 2 | 登录页 UI | 有了路由才能放登录页，且登录页不依赖其他新模块 |
| 3 | 认证上下文 + 路由守卫 | 登录页就绪后才能接入认证逻辑 |
| 4 | HeaderToolbar 扩展 | 认证就绪后才能显示用户信息和登出 |
| 5 | 「结束并记录通话」按钮 | 与认证无关但需要谨慎处理，放在认证稳定后做 |
| 6 | 通话状态选择弹窗 | 依赖任务包 5 的按钮触发 |
| 7 | 管理后台页面框架 | 依赖路由和认证，且是前端最后一块拼图 |

## 4. 高风险改动提醒

| 风险点 | 具体问题 | 防范措施 |
|--------|---------|---------|
| **「重置」与「记录」误绑定** | 如果「重置对话」的事件链中意外触发了通话记录写入，会产生大量脏数据 | 两个按钮必须走完全独立的回调链；「重置对话」回调中不得出现任何与记录相关的逻辑 |
| **App.tsx 重构范围控制** | 引入路由时 App.tsx 需要从"组合层"变为"路由入口"，如果不小心会破坏现有组件的 props 传递 | 将现有 App.tsx 内容平移到 AppPage.tsx，App.tsx 只做路由分发 |
| **HeaderToolbar props 变更** | 新增 props 如果改了接口签名，可能导致现有调用方报错 | 新增的 props 全部设为可选（`?`），不影响现有调用 |
| **Token 丢失导致页面白屏** | 认证逻辑如果在 Token 过期时没有正确处理，可能出现空白页 | ProtectedRoute 检测到无 Token 时立即跳转 `/login`，不渲染空内容 |
| **localStorage 键名冲突** | 认证 Token 使用 localStorage 存储，可能与现有话术数据的键名冲突 | Token 键名使用明确前缀如 `auth_token`，与现有 `existingScript` 等键名不冲突 |

## 5. 本阶段建议暂缓的前端改动

| 暂缓内容 | 原因 |
|---------|------|
| 改造 useScriptManager 数据源 | 第一阶段话术数据继续用 localStorage，不改为 API |
| 移除 vite-plugin-singlefile | 不影响在线版运行，移除可能影响离线版兜底，等在线版稳定后再处理 |
| 重构 storage.ts | 现有模块稳定，第一阶段无需改动 |
| 管理后台统计图表 | 第一阶段用纯表格展示即可，不引入图表库 |
| 话术在线编辑 | 继续用 CSV 上传，不做在线编辑器 |
| 普通用户个人统计页 | 第一阶段统计仅管理端可见 |

---

## 6. 前端实施结论

1. **共 7 个任务包**，按路由 → 登录 → 认证 → 工具栏 → 记录按钮 → 状态弹窗 → 管理后台的顺序逐步实施
2. **现有 12 个零改动复用的文件不动**，只有 `main.tsx`、`App.tsx`、`HeaderToolbar.tsx`、`ChatPanel.tsx` 需要最小扩展
3. **「结束并记录通话」和「重置对话」必须完全隔离**，这是前端改造中最容易出错的点
4. **每个任务包完成后独立验收**，确认不破坏现有功能后再推进下一个
5. **话术数据管理不改造**，第一阶段保持 localStorage 方式不变
