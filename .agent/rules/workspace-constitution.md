# 智慧外呼助手工作区规则

## 项目定位
这是一个面向汽车 4S 店销售顾问使用的智慧外呼助手项目。
当前项目已经有可运行的 React + TypeScript 版本，核心能力已经存在。

## 当前阶段目标
当前阶段不是重做项目，而是在现有项目基础上，逐步完成在线化迁移。

## 必须遵守的原则
1. 优先复用现有代码和模块，不要无必要重写
2. 每次做较大改动前，先说明理解和方案
3. 优先做最小可用版本，再逐步扩展
4. 所有界面文案默认使用中文
5. 不要擅自改变已有核心业务逻辑
6. 对数据表、字段、统计口径的新增，先解释用途再实施
7. 所有输出要尽量结构清晰，便于人工审核

## 已知业务事实
1. 当前项目有两个主要场景：
   - 首次邀约
   - 保有潜客
2. 当前项目已有核心能力：
   - 通话流程模拟
   - FAQ 三级菜单查询
   - CSV 话术上传/下载
3. 后续要逐步增加的能力包括：
   - 在线部署
   - 用户登录
   - 门店/人员映射
   - 通话记录
   - 数据统计
   - 管理后台
   - 自动日报/邮件回传

## 开发要求
1. 所有实现尽量保持简单、清晰、可维护
2. 不要为了"看起来高级"而过度设计
3. 先保证能跑通，再考虑优化
4. 修改代码后，需给出：
   - 本次改了什么
   - 为什么这样改
   - 如何验证
   - 有什么风险
