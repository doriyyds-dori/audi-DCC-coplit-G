# 智慧外呼助手在线版第一阶段范围说明

> 日期：2026-03-24

---

## 1. 第一阶段目标

**让 400 名销售顾问通过网址登录使用话术助手，每次通话结束后记录通话状态，管理员可在后台查看使用数据。**

---

## 2. 第一阶段必须包含的功能

1. **用户登录**：输入用户名密码登录，身份关联到门店和人员
2. **话术模拟**：登录后使用现有通话流程模拟 + FAQ 查询（复用现有功能）
3. **通话状态标记**：每次通话结束后必须选择状态（未接通 / 无意向 / 有意向），选完才算一次完整通话
4. **通话记录存储**：每次完整通话写入服务端数据库，不依赖 localStorage
5. **管理后台（基础版）**：管理员可查看每日各用户的通话次数和状态分布
6. **在线部署**：部署到国内可访问的服务，400 人可同时使用

---

## 3. 第一阶段明确不做的功能

| 不做 | 原因 |
|------|------|
| 自动邮件回传 | 优先把数据采集和查看跑通，邮件是锦上添花 |
| Excel 自动导出 | 第一阶段先在后台页面直接看数据 |
| 用户自助注册 | 400 人账号由管理员统一创建 |
| 话术数据在线管理 | 继续用 CSV 上传，不做在线编辑器 |
| 离线版打包 | 在线版上线后离线版不再维护 |
| 多角色权限体系 | 第一阶段只分管理员和普通用户两种角色 |

---

## 4. 前台用户最小使用流程

1. 打开网址（由管理员分发链接）
2. 输入用户名和密码登录
3. 选择客户类型（保有潜客 / 首次邀约）
4. 点击「开始邀约」，进入通话模拟
5. 根据客户实际反馈，点选对应按钮查看话术；如遇突发问题，使用右侧 FAQ 菜单
6. 通话结束后，弹出状态选择：未接通 / 明确无意向 / 有意向
7. 选择状态后，本次通话记录自动保存
8. 继续下一通电话，或关闭页面

---

## 5. 管理端最小能力

1. **查看今日统计**：按用户显示当日通话总数、各状态数量
2. **查看历史统计**：选择日期查看指定日期数据
3. **用户管理**：新增用户（指定用户名、密码、所属门店）、禁用用户
4. **门店管理**：新增/编辑门店名称

不做：报表导出、图表可视化、权限细分、操作日志。

---

## 6. 数据最小闭环

### 必须记录的数据

| 表 | 最小字段 | 解决什么问题 |
|----|---------|------------|
| **users** | id, username, password_hash, store_id, role, enabled | 谁在用、属于哪个门店、能否登录 |
| **stores** | id, name | 门店归属，方便按门店汇总 |
| **call_records** | id, user_id, date, status, created_at | 谁在哪天打了多少电话、结果如何 |

### status 取值

| 值 | 含义 |
|----|------|
| `not_connected` | 未接通 |
| `no_intent` | 明确无意向 |
| `has_intent` | 有意向 |

### 这些数据回答的核心问题

- 每个人每天打了多少通电话？
- 接通率多少？
- 有意向客户占比多少？
- 哪个门店的使用频率最高/最低？

---

## 7. 第一阶段完成标准

满足以下全部条件，即视为第一阶段完成：

1. ✅ 400 名用户可通过网址登录，全国各地可正常访问
2. ✅ 登录后话术模拟功能与现有离线版一致
3. ✅ 每次通话结束后必须选择状态，选完后记录写入服务端
4. ✅ 管理员可在后台查看任意日期、任意用户的通话次数和状态分布
5. ✅ 管理员可新增和禁用用户
6. ✅ 数据不因用户关闭浏览器而丢失
